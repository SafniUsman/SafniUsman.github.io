Buildroot Integration and Customization

Project Overview: This project involves integrating and customizing Buildroot, a popular embedded Linux build system, for creating a tailored system image. The primary goal is to automate the process of building a customized Linux system with specific packages and configurations, ensuring efficiency, consistency, and reproducibility.

GitHub Link to the project: Buildroot Integration

Key Steps and Components:
1. Buildroot Submodule Integration:
• Added Buildroot as a git submodule from the official repository (branch 2022.02.x) to the project structure.
• Ensured proper commit hash management for Buildroot to maintain a specific version.
2. Package Integration:
• Incorporated mandatory packages as per Buildroot documentation.
• Developed a custom package, aesd-assignments, that builds applications based on the code from Assignment 3 and later. This package includes a tester script, cross-compiled applications, and dependencies.
• Implemented package configurations in Config.in, external.mk, and external.desc files.
3. Configuration Management:
• Configured Buildroot using qemu_aarch64_virt_defconfig.
• Utilized save-config.sh to save project-specific configurations, ensuring consistency.
4. Build Process:
• Executed the Buildroot build process using ./build.sh, emphasizing the importance of initial and subsequent builds.
• Applied suggested speedup techniques to enhance build efficiency.
5. QEMU Image Execution:
• Deployed the generated QEMU image using runqemu.sh script, facilitating efficient testing and validation.
6. System Enhancement:
• Added the Dropbear package to the image to support SSH for secure communication.
• Configured default root password and SSH access for enhanced system security.
7. Verification and Testing:
• Validated the system by logging in via SSH, ensuring successful integration and functionality.
• Utilized SCP to transfer essential files from the QEMU instance to the assignment repository.
8. Clean-Up Script:
• Implemented a clean-up script (clean.sh) to reset the Buildroot environment for future developments.
