Manual Linux kernel and root filesystem

Project Overview: The project focuses on generating a minimalistic kernel and root filesystem for embedded systems development. By leveraging an ARM cross-compile toolchain and QEMU for testing, the goal is to create a flexible solution that allows customization based on diverse project requirements.

GitHub Link to the project: Manual linux kernel Refer finder-app/manual-linux.sh

Key Steps and Components:
1. Script Overview:
• The core component is the BASH script "manual-linux.sh."
• Took an argument (outdir) for specifying the output location.
2. Directory Creation:
• Checked and created the specified outdir directory, failing gracefully if unable to create.
3. Kernel Image Building:
• Fetched Linux kernel source code from the official repository.
• Utilized an ARM cross-compile toolchain for kernel compilation.
4. Copying Resulting Files:
• Copied the kernel image and related files to the specified outdir.
• Key output files: outdir/Image (kernel image) and outdir/initramfs.cpio.gz (root filesystem).
5. Root Filesystem Building:
• Constructed a root filesystem in outdir/rootfs based on a configurable staging directory tree.
6. Testing with QEMU:
• Utilized QEMU for testing the generated kernel and root filesystem in a virtualized environment.

Project outcome: The implemented BASH script, "manual-linux.sh," streamlines the process of kernel and root filesystem generation, providing a user-friendly and efficient solution for embedded systems development. The script successfully fetches the Linux kernel source code from the stable repository and employs an ARM cross-compile toolchain to compile the kernel. This process results in a functional and optimized kernel image suitable for embedded systems. The project excels in constructing a root filesystem in the designated outdir/rootfs directory. This filesystem is derived from a staging directory tree, providing a foundation for embedding essential components tailored to project needs.
