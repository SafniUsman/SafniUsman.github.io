Socket-based data transfer Server

Project Overview: This project focuses on the development of a socket-based server application, named aesdsocket, capable of receiving data packets, appending them to a file, and responding to client requests. The server operates as a daemon and supports cross-compilation, enhancing its versatility for embedded systems. Additionally, the implementation ensures robust error handling, graceful exits, and seamless integration with system signals.

GitHub Link to the project: Socket Server

Key Steps and Components:
1. Directory Modification:
• Extended the existing assignments-3-and-later repository by adding a new directory named "server."
2. Socket Server Implementation:
• Created a Makefile in the "server" directory, enabling compilation through the "all" and "default" targets.
• Developed the aesdsocket program, a socket-based server, supporting cross-compilation and producing an executable in the "server" directory.
3. Socket Operations:
• Established a stream socket bound to port 9000.
• Listened for and accepted connections from clients.
• Logged connection information, including the client's IP address, to the system syslog.
4. Data Handling:
• Received data packets over the connection, delimited by newline characters.
• Appended received data to the file /var/tmp/aesdsocketdata, creating the file if it does not exist.
• Handled potential errors related to malloc() for packet handling.
5. Response to Clients:
• Returned the full content of /var/tmp/aesdsocketdata to the client as soon as a data packet completed.
6. Graceful Shutdown:
• Logged closure information, including the client's IP address, to the system syslog.
• Implemented a loop for continuously accepting connections until a SIGINT or SIGTERM signal is received.
7. Signal Handling:
• Gracefully exited the application upon receiving SIGINT or SIGTERM signals.
• Completed open connection operations, closed sockets, and deleted the file /var/tmp/aesdsocketdata.
• Logged a message to the system syslog indicating the reception of a signal: "Caught signal, exiting."
8. Daemon Mode:
• Enhanced the application to support a -d argument, enabling it to run as a daemon.
• Forked the process after ensuring successful binding to port 9000 in daemon mode.
9. Integration Testing:
• Installed the netcat utility on the development system for testing purposes.
• Utilized the sockettest.sh script to validate the functionality of the server, ensuring successful completion after restarting the application.
10. Full Test Automation:
• Modified the program to support daemon mode for comprehensive testing.
• Integrated the ./full-test.sh script from the aesd-assignments repository to automate the verification process.
